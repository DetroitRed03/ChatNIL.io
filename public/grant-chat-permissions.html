This page will help you grant the necessary permissions for the chat persistence feature.

Problem: The API routes are getting "permission denied for table chat_sessions" errors because the service_role doesn't have GRANT permissions on the chat tables.

Option 1: Copy SQL and Run in Supabase Dashboard (Recommended)

Copy the SQL below and paste it into your Supabase SQL Editor:

Copied! ‚úì
-- Grant ALL permissions on chat tables to service_role
GRANT ALL ON chat_sessions TO service_role;
GRANT ALL ON chat_messages TO service_role;

-- Grant sequence permissions
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO service_role;

-- Verify the grants worked
SELECT grantee, table_name, privilege_type
FROM information_schema.role_table_grants
WHERE table_name IN ('chat_sessions', 'chat_messages')
AND grantee = 'service_role';
  1. Click "Copy SQL to Clipboard" button above
  2. Go to Supabase SQL Editor
  3. Paste the SQL and click "Run"
  4. You should see a table showing the granted permissions
  5. Return here and click "Test API" below

Option 2: Run via Supabase API (Experimental)

This will attempt to execute the SQL using the Supabase API with your service role key.

Note: This requires your service role key to be in .env.local file.
